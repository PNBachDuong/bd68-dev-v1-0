# Impeccable Design Compass

Use this reference before frontend brainstorming or design planning.

Source:
- https://github.com/pbakaus/impeccable
- https://raw.githubusercontent.com/pbakaus/impeccable/main/source/skills/frontend-design/SKILL.md

## Why It Matters
- Impeccable builds on Anthropic's `frontend-design` skill with deeper design steering, domain references, anti-patterns, and command-style review loops.
- It is useful as a pre-plan quality filter, especially for WildSoul UI or any design-led feature.

## Pre-Brainstorm Checklist
- Define purpose, audience, and constraints first.
- Commit to one bold aesthetic direction instead of mixing several moods.
- Decide what the screen should be remembered for.
- Reject ideas that already look like generic AI UI.

## Anti-Patterns To Screen Out
- Overused fonts like Inter, Arial, Roboto, or system defaults.
- Purple-blue gradient defaults, cyan-on-dark palettes, or glow-heavy dark mode.
- Gray text on colored backgrounds.
- Card-inside-card layouts and repetitive icon-heading-text card grids.
- Generic metric-hero layouts or centered-everything compositions.
- Decorative motion without purpose.

## What To Borrow Into Planning
- Typography should be intentional and characterful.
- Colors should feel designed, not defaulted.
- Layout should create rhythm, asymmetry, and hierarchy.
- Motion should support state and emphasis, not spectacle for its own sake.
- UX writing should remove redundancy and make each word earn its place.

## Planning Output Rule
When using this compass, keep the planning output short:
- aesthetic direction
- 2-4 non-negotiable design rules
- key anti-patterns to avoid
- one memorable differentiator

Do not dump long style guides unless the user explicitly asks for them.
